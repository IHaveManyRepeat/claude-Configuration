# 安全概述

> 本文档定义了代码开发过程中需要遵守的安全原则和检查项，确保系统具备高安全性。

## 核心安全原则

### 1. 纵深防御

- 多层安全防护，而非依赖单一安全机制
- 每个层面都应进行安全检查
- 遵循"最小权限"原则

### 2. 默认安全

- 默认配置应为最安全状态
- 明确禁用不必要的功能
- 安全特性应默认开启

### 3. 失败安全

- 安全检查失败时默认拒绝访问
- 错误消息不泄露敏感信息
- 异常情况下确保系统处于安全状态

## 代码安全检查清单

### 输入验证

- [ ] 所有用户输入必须验证，不信任任何外部数据
- [ ] 使用白名单验证而非黑名单
- [ ] 验证数据类型、长度、格式、范围
- [ ] 防止 SQL 注入：使用参数化查询
- [ ] 防止 XSS：对输出进行转义
- [ ] 防止命令注入：避免直接执行用户输入

### 认证与授权

- [ ] 密码存储使用强哈希算法（如 bcrypt、argon2）
- [ ] 敏感操作需要二次验证
- [ ] 实现基于角色的访问控制（RBAC）
- [ ] 会话令牌安全生成和存储
- [ ] 防止暴力破解：实现账户锁定或验证码

### 敏感数据保护

- [ ] 敏感数据加密存储
- [ ] 敏感数据传输使用 TLS
- [ ] 禁止硬编码密钥/密码/令牌
- [ ] 使用环境变量或密钥管理器存储 secrets
- [ ] 日志中不记录敏感信息
- [ ] 脱敏处理返回的用户数据

### 错误处理

- [ ] 统一错误处理机制
- [ ] 错误消息不泄露系统细节
- [ ] 记录详细错误日志（用于审计）
- [ ] 避免信息泄露：堆栈跟踪不上生产

### API 安全

- [ ] 所有 API 端点进行身份验证
- [ ] 实现速率限制防止滥用
- [ ] 使用 CSRF 令牌防护
- [ ] CORS 配置严格控制
- [ ] 响应头安全设置（HSTS、X-Frame-Options 等）

## 常见安全漏洞检查

### OWASP Top 10 重点关注

1. **注入攻击** - 参数化查询、输入过滤
2. **身份验证失效** - 强密码策略、会话管理
3. **敏感数据泄露** - 加密传输、加密存储
4. **XML 外部实体** - 禁用 XXE
5. **访问控制失效** - 最小权限原则
6. **安全配置错误** - 最小化暴露
7. **跨站脚本 (XSS)** - 输入转义、输出编码
8. **不安全的反序列化** - 验证来源、限制类型
9. **使用有漏洞的组件** - 定期更新依赖
10. **日志记录不足** - 完整审计日志

## 安全开发流程

```
1. 需求阶段
   └── 识别安全需求 → 威胁建模

2. 设计阶段
   └── 安全架构设计 → 安全评审

3. 开发阶段
   └── 遵循安全编码规范 → 静态代码分析

4. 测试阶段
   └── 安全测试 → 渗透测试

5. 部署阶段
   └── 安全配置 → 漏洞扫描
```
