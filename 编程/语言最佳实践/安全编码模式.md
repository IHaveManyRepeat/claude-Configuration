## 通用安全原则

### 输入验证

```plaintext
┌─────────────────────────────────────────────────────────┐
│  永远不要信任用户输入                                      │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│  • 验证所有来自外部的数据源                                │
│  • 使用白名单验证策略                                      │
│  • 验证数据类型、长度、格式、范围                          │
│  • 在边界进行验证（API 入口、数据库查询）                  │
└─────────────────────────────────────────────────────────┘
```

### 敏感数据保护

```plaintext
┌─────────────────────────────────────────────────────────┐
│  敏感数据处理                                            │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│  • 禁止硬编码密钥/密码/令牌                               │
│  • 使用环境变量或密钥管理器                               │
│  • 传输使用 TLS，存储使用加密                            │
│  • 日志中不记录敏感信息                                   │
└─────────────────────────────────────────────────────────┘
```

### 认证与授权

```plaintext
┌─────────────────────────────────────────────────────────┐
│  身份验证最佳实践                                         │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│  • 密码使用强哈希算法（bcrypt/argon2）                   │
│  • 实现最小权限原则                                       │
│  • 敏感操作需要二次验证                                   │
│  • 防止暴力破解：账户锁定/验证码                          │
└─────────────────────────────────────────────────────────┘
```

## 各语言安全实践

### Go

详见：[[Go 安全编码模式]]

核心关注点：
- SQL 注入防护（使用 sqlx 或 database/sql 参数化查询）
- 模板注入防护（html/template 自动转义）
- 密钥管理（使用环境变量或 vault）
- 并发安全（sync 包正确使用）

### Python

详见：[[Python 安全编码模式]]

核心关注点：
- 输入验证（Pydantic 模型验证）
- SQL 注入防护（使用 ORM 或参数化查询）
- 序列化安全（避免 pickle 反序列化不可信数据）
- 依赖安全（定期 audit）

### JavaScript

详见：[[JavaScript 安全编码模式]]

核心关注点：
- XSS 防护（输出转义、内容安全策略 CSP）
- 命令注入防护（避免 eval、child_process）
- 敏感数据保护（不暴露到前端）
- 模块安全（npm audit）

### TypeScript

详见：[[TypeScript 安全编码模式]]

核心关注点：
- 类型安全作为安全屏障
- 输入验证（zod/class-validator）
- XSS 防护（同 JavaScript）
- API 安全（rate limiting、CSRF）

### Java

详见：[[Java 安全编码模式]]

核心关注点：
- SQL 注入防护（PreparedStatement）
- XSS 防护（ESAPI、Spring Security）
- 序列化安全（ObjectInputStream 白名单）
- 认证框架（Spring Security）

### Rust

详见：[[Rust 安全编码模式]]

核心关注点：
- 内存安全（所有权系统天然防护）
- 错误处理（Result 模式）
- 依赖安全（cargo audit、cargo deny）
- 并发安全（严格的借用检查）

## 安全检查清单

开发自检：
- [ ] 无硬编码密钥
- [ ] 用户输入已验证
- [ ] SQL 使用参数化查询
- [ ] 输出已转义
- [ ] 敏感数据不记录日志
- [ ] 依赖已审计
- [ ] 错误消息不泄露系统信息
