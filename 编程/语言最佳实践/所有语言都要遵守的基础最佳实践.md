# 基础编码最佳实践

> 源自《重构》

---

## 1. 命名自解释

清晰、准确、不玩缩写，命名本身就能说明意图，无需注释。

```go
// 坏：u c m 鬼知道是什么
func doSomething(u *User, c *Conv, m string) error

// 好：直接看懂
func SendMessageToConversation(user *User, conversation *Conversation, messageContent string) error
```

前端同理：`ConversationListComponent` 优于 `Comp`，`handleSendMessageClick` 优于 `handleClick`

---

## 2. 注释

**只写"为什么"，不写"怎么做"**。如果需要注释，先重构代码让它自解释。

```go
// 好：只写原因
// 这里必须用事务，因为创建会话和初始化第一条消息必须同时成功或同时失败
tx := db.Begin()
```

---

## 3. 组合优于继承

**用接口 + 组合，不用继承**。

```go
// 坏：继承体系僵化
type OpenAIModel struct { BaseModel }

// 好：接口 + 组合
type ModelAdapter interface {
    ChatCompletion(ctx context.Context, req *ChatRequest) (<-chan *ChatResponseChunk, error)
}
type OpenAIAdapter struct { client *http.Client; apiKey string }
```

前端：用 Hooks、用组合组件，不用 `BaseComponent` 继承

---

## 4. 不可变优于可变

**优先使用不可变数据，避免意外副作用**。

```go
// 坏：修改原对象
func addItem(items []string, item string) []string {
    items = append(items, item)  // 突变
    return items
}

// 好：返回新对象
func addItem(items []string, item string) []string {
    newItems := make([]string, len(items)+1)
    copy(newItems, items)
    newItems[len(items)] = item
    return newItems
}
```

---

## 5. 函数单一原则 (SRP)

**短小、单一职责、不超过一屏**。一个函数只做一件事。

```go
// 坏：一个函数干所有事
func HandleChatRequest(w http.ResponseWriter, r *http.Request) { /* 100行... */ }

// 好：拆成小函数
func HandleChatRequest(w http.ResponseWriter, r *http.Request) {
    user := authMiddleware.GetUser(r)
    conv := conversationService.GetCurrentConversation(user)
    msg := messageService.SaveUserMessage(conv, r.Body)
    responseCh := modelGateway.Chat(conv, msg)
    streamResponse(w, responseCh)
}
```

## 6. 错误处理 + 失败快速

**不吞错误，带上下文；在边界提前校验参数，立即报错**。

```go
// 坏：犯罪
result, _ := modelGateway.CallModel(ctx, req)

// 好：清晰、带上下文
result, err := modelGateway.CallModel(ctx, req)
if err != nil {
    log.Errorw("model call failed", "trace_id", traceID, "user_id", userID, "error", err)
    return fmt.Errorf("call model %s: %w", modelName, err)
}

// 参数校验
if user == nil {
    return nil, fmt.Errorf("user is required")
}
```

前端：别 `try {} catch (e) {}` 空捕获，要给用户提示 + 上报日志

---

## 7. DRY 原则

**禁止重复，三次重复必须抽象复用**。

```go
// 坏：到处复制粘贴
log.Infow("conversation created", "user_id", userID, "conv_id", convID)

// 好：提取公共函数
func LogWithCtx(ctx context.Context, level string, msg string, keysAndValues ...interface{})
```

---


## 8. 代码整洁

**小步重构，保持代码随时可提交**。

---

## 9. 测试

**没有测试就不敢重构。测试是重构的安全网。覆盖率目标为所有代码必须是路径覆盖100%的标准**。

---

## 10. 安全

**在系统边界严格校验所有输入，永不信任用户数据**。防止注入、越权、泄露。

---

## 11. 性能

**先写对，再优化；别过早优化**。
